重启RN计划

发表于 2019-05-23

选择脚手架

CRN 由携程无线平台研发团队基于React Native框架优化，定制成稳定性和性能更佳、也更适合业务场景的的跨平台开发框架。

开源基于ReactNative 0.59.0, react 16.8.3版本, 开源的主要是性能优化部分, 也是规模化使用RN进行业务开发必须要做的优化

将RN项目打包成JSBundle

在进行原生开发的时候，通常我们需要RN项目打包成Bundle文件，然后由原生来进行加载。

打包命令。

使用react-native bundle –help 来查看打包的具体参数。

react-native bundle[参数]构建js离线包。

在RN项目根目录下创建/out/ios文件夹和/out/android文件夹。

IOS

Android

我们在打包bundle的时候，默认使用模块名为RN项目工程中的app.json的name，在原生代码引用这个bundle时，这个moduleName要保持一致。

如果需要修改模块名，则对应的rn工程里，package.json/app.json的name需要修改，并且，在android文件夹中，修改MainActivity的getMainComponentName()方法返回的name，在ios文件夹中，修改AppDelegate.m的moduleName。

如果出现无法打安卓包的情况，考虑是否正确安装配置了JDK；如果出现无法写文件的情况，mac或者linux系统在命令前添加sudo。
